# Migration Guide

This document collects upgrade notes for breaking changes between major versions of `electrolux-to-mqtt`.

Conventions:

- A breaking change is any commit whose Conventional Commit type carries the `!` marker (e.g. `feat!:`, `fix!:`, `refactor!:`). See `.claude/CLAUDE.md` § Tooling.
- Each section below covers one major version. Within a section, list the user-visible breaks and the steps to adapt.
- For non-breaking changes, see [`CHANGELOG.md`](./CHANGELOG.md) (generated by semantic-release on each merge to `main`).
