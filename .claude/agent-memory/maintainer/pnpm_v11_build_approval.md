---
name: pnpm v11 build script approval via pnpm-workspace.yaml
description: pnpm v11 uses allowBuilds in pnpm-workspace.yaml for build script approval; onlyBuiltDependencies in package.json is the v10 mechanism
type: project
---

pnpm v11 introduced `allowBuilds` in `pnpm-workspace.yaml` as the new mechanism for approving build scripts. Without it, `ERR_PNPM_IGNORED_BUILDS` is emitted on every install.

The project has both mechanisms present:
- `pnpm-workspace.yaml`: `allowBuilds: { esbuild: true, "@biomejs/biome": true }` (v11, workspace-level)
- `package.json` `pnpm.onlyBuiltDependencies`: `["@biomejs/biome", "esbuild"]` (v10 style, kept for compatibility)

`pnpm-workspace.yaml` was created during the pnpm v10→v11 store migration in May 2026 to resolve the build script warning. The `allowBuilds` format is a boolean map, not an array: `esbuild: true`.

**Why:** pnpm v11 changed build approval from per-package `onlyBuiltDependencies` to workspace-level `allowBuilds`. The placeholder generated by `pnpm approve-builds` (`esbuild: set this to true or false`) must be replaced with the actual boolean.

**How to apply:** If `ERR_PNPM_IGNORED_BUILDS` appears after a pnpm upgrade, check `pnpm-workspace.yaml` — ensure `allowBuilds` has `true` values, not the placeholder string.
