name: pnpm v11 build script approval via pnpm-workspace.yaml
description: pnpm v11 uses allowBuilds in pnpm-workspace.yaml for build script approval; onlyBuiltDependencies is the removed v10 mechanism
type: project
---

pnpm v11 introduced `allowBuilds` in `pnpm-workspace.yaml` as the new mechanism for approving build scripts. Without it, `ERR_PNPM_IGNORED_BUILDS` is emitted on every install.

The project uses `pnpm-workspace.yaml` exclusively — no `pnpm.onlyBuiltDependencies` fallbacks anywhere:
- `pnpm-workspace.yaml` (root): `allowBuilds: { esbuild: true, "@biomejs/biome": true }` — covers root installs
- `telemetry-backend/pnpm-workspace.yaml`: `allowBuilds: { esbuild: true }` — required because telemetry-backend is NOT a workspace member; pnpm v11 detects the root workspace file and short-circuits `pnpm install` in subdirectories without their own workspace file, skipping local node_modules creation entirely

`pnpm-workspace.yaml` was created during the pnpm v10→v11 store migration in May 2026. `telemetry-backend/pnpm-workspace.yaml` was added shortly after to fix CI's standalone `cd telemetry-backend && pnpm install` path.

**Why:** pnpm v11 changed build approval from per-package `onlyBuiltDependencies` to workspace-level `allowBuilds`. The placeholder generated by `pnpm approve-builds` (`esbuild: set this to true or false`) must be replaced with the actual boolean.

**How to apply:** If `ERR_PNPM_IGNORED_BUILDS` appears after a pnpm upgrade, check both `pnpm-workspace.yaml` files — ensure `allowBuilds` has `true` values. Do NOT restore `pnpm.onlyBuiltDependencies` — user explicitly removed all v10 fallbacks.
